# AIO-Scope 使い方ガイド (営業部員向け)

所要時間: 初回15分で読める / 実案件での初回実施は約2-3日

---

## TL;DR

1. URL を開く → ログイン
2. 「新規案件を作成」→ 案件情報を入力
3. 6つのタブを順番に埋める (約2-3日)
4. 「📥 エクスポート」から Excel / PPT をダウンロード
5. クライアントMTG で提示
6. **必ず「JSONバックアップ」をダウンロードして Google Drive に保存**

---

## Step 1: ログイン

1. AIO-Scope URL にアクセス
2. メールアドレス (`yourname@goodpatch.com`) + 共通パスワードを入力
3. 「ログイン」をクリック

## Step 2: 新規案件を作成

ログイン後のトップ画面で「新規案件を作成」。以下を入力:

| 項目 | 例 | 必須 |
|------|-----|------|
| 案件名 | 三井ダイレクト損保リニューアル | ✓ |
| クライアント名 | 三井ダイレクト損害保険株式会社 | ✓ |
| 業界カテゴリ | ダイレクト自動車保険 | |
| 対象URL | https://www.mitsui-direct.co.jp/ | |
| 競合社 (1行1社) | ソニー損保\nイーデザイン損保\nSBI損保 | |

「案件を作成して診断開始」を押すと、診断ウィザードが始まります。

## Step 3: 6領域の診断を進める

画面上部のタブを順番にこなします。

### ① AI引用 (所要: 約30分 ← 自動採点で短縮)

15クエリ × 2AI の入力。**完全手動ではなく、応答ペーストで半自動採点** です。

**推奨ワークフロー (1クエリあたり約2分)**:

1. 各クエリのカードを開く → 置換後のクエリが表示されます
2. **🚀 ChatGPTで開く** / **🚀 Perplexityで開く** ボタンで各AIを起動
   - 両方ともクエリ自動入力対応。ワンクリックで新タブに入力済みで開きます
   - (Claude/Geminiは手動ペーストが必要なためv1.2で除外)
3. 各AIの応答を **そのままテキストエリアに貼り付け**
4. 自動で **引用/位置/正確性** のスコアが提案される
5. 「⚡ この提案を採用」をクリック → 値が反映される
6. 違和感があれば手動で微調整

**自動判定の仕組み**:
- クライアント名がテキストに登場するか → 引用 (0/1)
- テキスト内の位置 (先頭/中盤/末尾) → 位置 (3/2/1)
- 登場回数 + 特徴的キーワード有無 → 正確性の提案値

**APIは一切使わず、ローカル文字列マッチングで動作** します。月額費用ゼロ。

### ② Google SGE (所要: 約20分)

20キーワード分の行があります。各行に **🔎 Google検索** ボタンがあり、1クリックで新タブ検索。

1. キーワードを入力 (空欄でもOK)
2. 「🔎 Google検索」でシークレット推奨
3. AI Overview が表示されるか / 自社が引用されているか / リンクが露出するか の3つのチェックボックスを記入

### ③ 構造化データ (所要: 約20分)

アプリ内のボタンから直接ツールを起動できます:
- **🔍 Google Rich Results Test** - 対象URL込みで自動起動
- **🔍 Schema.org Validator** - 同上

1. ボタンから検査ツールを開く
2. 9項目それぞれを選択: なし / 部分的 / 完全

### ④ E-E-A-T (所要: 約20分)

1. サイトの著者ページ・会社概要・引用文献などを目視確認
2. 各項目を 0〜配点満点 のスライダーで採点

### ⑤ テクニカルSEO (所要: 約20分)

「⚡ PageSpeed Insights で測定」ボタンで対象URL込みでツールを起動。Core Web Vitalsや各種SEO指標を確認して採点。

### ⑥ 競合ベンチマーク (所要: 約45分)

競合3-5社について、領域1-5の合計スコアを入力。

**簡易確認の時短tip**: 各競合の展開カード内に「🚀 [AI名]」ボタンがあるので、「{競合名} {業界} の特徴・強み・評判」クエリを1クリックでAIに投げられます。全AI × 全クエリを回す必要はなく、代表的な1-2クエリで感覚値をつかめば十分。

## Step 4: ダッシュボードで可視化

「📊 ダッシュボード」タブで総合スコア・レーダーチャート・ランクを確認。

- **自社** (紫) vs **競合平均** (赤点線) が重ね表示される
- 弱い領域が視覚的に分かる

## Step 5: 所見を言語化

「✍️ 所見」タブで強み・弱み・改善アクションを書き出す。

### 強み・弱みは3-5項目ずつ

診断データを見て、「何が良かったか」「何が足りないか」を箇条書きで:

**例 (強み)**:
- Organization Schemaが完全実装、ブランド指名で業界平均以上の引用率
- Core Web Vitals全緑、テクニカル基盤は堅牢

**例 (弱み)**:
- FAQ/HowTo Schemaが未実装
- Perplexity/Geminiでのブランド認知が弱い
- 著者情報が未整備

### 改善アクションは3-8施策

「Impact」「Effort」「Period」を選んで優先度を整理:
- **Impact**: H/M/L (効果の大きさ)
- **Effort**: H/M/L (実装難易度)
- **Period**: Quick Win / 中期 / 長期

## Step 6: エクスポート

「📥 エクスポート」タブ:

1. **Excel**: スコアリング明細・全領域データを9シートで出力 → 社内ナレッジ用
2. **PowerPoint**: クライアント向け8スライド報告書 → MTGで提示
3. **JSON**: バックアップ (**必ず毎回ダウンロード**)

### 📌 重要: JSONバックアップの保存先

**Google Drive の [プロジェクトフォルダ](https://drive.google.com/drive/u/0/folders/1aQOBmIrxmSIZ6LvTQLXT2IFJXxdtsnrl) に保存してください**。

- セッションが切れるとアプリ内のデータは消えます
- JSONをアップロードし直すと続きから再開できます
- チーム共有も Google Drive 経由で可能

## Step 7: クライアント提示

MTGでの提示時間の目安: 15-20分

1. **オープニング (1分)**: 「独自に診断を実施しました」
2. **総合スコア (2分)**: スコアとランクを提示
3. **強み・弱み (5分)**: ギャップ分析スライド
4. **Quick Win (5分)**: 改善アクション、特に短期実行のもの
5. **提案への橋渡し (2分)**: 「これを提案書のUI/UX設計と並走して反映します」

## よくある質問 (FAQ)

### Q. 途中で疲れたら一旦保存したい
A. 左サイドバーの「💾 JSONエクスポート」で保存 → ブラウザを閉じる → 次回ログイン時にファイルアップロードで再開

### Q. チームで分担して進めたい
A. 領域ごとに担当を分け、それぞれが JSONをエクスポート → 統合は後で…は現状手動。v2で対応予定

### Q. 20キーワード・15クエリも時間がない
A. 以下のショートカットで時短可:
- 領域1 (AI引用): 15→8クエリに減らす
- 領域2 (SGE): 20→10キーワードに減らす
- 領域6 (競合): 3社に絞る、AI引用は各社5クエリ

### Q. クライアントが「なぜこのスコアなのか」を聞いてきた
A. Excel の該当シートに採点明細があるので、必要に応じて共有。全採点プロセスが透明

### Q. パスワードを忘れた
A. Market Design Div の管理者 (田原さん) まで Slack DM

### Q. URL に社外からアクセスできる?
A. `@goodpatch.com` ドメインのメールアドレスが必須なので、社外者はログインできない仕組み
